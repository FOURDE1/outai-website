# Assets Folder

Place your images here following this naming convention:

## Hero Section
- `hero-phone-mockup.png` - Main phone mockup image (recommended: 400x800px PNG with transparency)
- `hero-phone-screen.png` - App screenshot for phone screen
- `hero-background.jpg` - Optional hero background image

## Features (Craft Your Journey)
- `feature-comfort.jpg` - Refined Comfort feature image
- `feature-service.jpg` - Seamless Service feature image
- `feature-ambiance.jpg` - Curated Ambiance feature image

## Services
- `service-rides-icon.svg` - Rides service icon
- `service-rental-icon.svg` - Daily Renting service icon
- `service-care-icon.svg` - Care service icon

## Blog Posts
- `blog-post-1.jpg` - Featured blog post image
- `blog-post-2.jpg` - Blog post thumbnail
- `blog-post-3.jpg` - Blog post thumbnail

## Misc
- `app-store-badge.svg` - App Store download badge
- `play-store-badge.svg` - Google Play download badge
- `team-photo.jpg` - Team or about section photo (if needed)

## Image Guidelines

1. **Optimization**: Compress all images before adding
   - Use WebP format when possible
   - PNGs for images with transparency
   - JPGs for photos

2. **Sizes**:
   - Hero images: Max 1920px wide
   - Feature images: 600-800px wide
   - Thumbnails: 400px wide
   - Icons: 64x64 or 128x128 SVG preferred

3. **Naming**:
   - Use lowercase
   - Use hyphens, not spaces
   - Be descriptive: `section-description-variant.ext`

4. **Import in Code**:
```tsx
import heroMockup from '@/assets/hero-phone-mockup.png';
```
